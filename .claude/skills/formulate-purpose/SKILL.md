# /formulate-purpose — 種を反証可能な目的仮説に精錬する

`/surface` で出た種や曖昧な思いつきを、確信度とステータスを持つ**反証可能な目的仮説**に変える。**一度に1問ずつ**深掘りする。

> **共通規約**（案件解決・ID/接頭辞・リンク記法と `../` 深さ・.gitkeep・承認規律）は [CLAUDE.md「スキル共通規約」](../../../CLAUDE.md) が正典。以下は本スキル固有の手順。

## 手順

1. **現在案件を読む**（→ 共通規約1）。[playbooks/formation.md](../../../playbooks/formation.md) の「問いかけバンク B（反証可能化）」を読む。`wiki/index.md` で対象の種／既存 P を把握する。

2. **1問ずつ深掘りする** — `AskUserQuestion` で問いかけバンク B を**1問ずつ**。特に「何が起きたら“やっぱり違った”と分かるか（反証条件）」「目的を果たしている具体的な一日」「今の手中の鳥（[[C-NNN]]）のどれから立ち上がるか」を詰める。回答が曖昧なら反証可能になるまでさらに1問。
   - ※ 当事者性温度（腰の入り方）での**出題分岐はフェーズ2**。フェーズ1では観測手がかりとして自由記述で拾うにとどめ、確信度・判定には使わない。

3. **反証可能な一文に整形する** — 「私は〈状況・制約〉のもとで〈誰／何〉のために〈こういう目的〉を果たしたい」の形式で、何が起きれば揺らぐ／立ち上がるかが明確な一文にする。検証不能な言い切り（「〜が大事だと思う」）は、何を試せば確かめられるかを問い返して書き換える。

4. **タイプ・初期確信度・ステータスを決める** — P にサブタイプは無い。証拠がまだ壁打ちだけなら確信度 1-2、探索を始めていれば `status: 探索中`。**外界の証拠がまだ無い段階では確信度を 3 以上に上げない**（証拠の非対称性。上げるのは `/reflect`）。核心の目的なら frontmatter `core: true`。

5. **レコードを発番・作成する** — `wiki/purposes/` の既存最大+1で `P-NNN` を採番（再利用禁止）。`templates/purpose.md` に従って作成。`id` はファイル名と同じ接頭辞つき。`short-title` を書く。
   - **接地**: 手中の鳥に根ざすなら frontmatter `grounded-in` に対応する制約 `[[C-NNN]]` を列挙し、本文「系譜・接地」にも wikilink を併記する（未接地なら `/hand` を促す）。
   - **系譜**: 種が別の P から派生・ピボットしたなら `derived-from`（単一）、因果的に導くなら `leads-to`、を frontmatter＋本文 wikilink の二重表現で書く。
   - 確信度履歴テーブルに初期行を入れる。`.gitkeep` が残っていれば削除してよい。

6. **index と log を更新する** — `wiki/index.md` の目的仮説表に1行追加。`wiki/log.md` に追記:
   `## [YYYY-MM-DD] purpose | P-NNN <要約> → 精錬・確信度N/ステータス`

## 目的仮説の取り下げ・書き換え（revises）

- **取り下げ**（試行に未紐づけの撤回。反証ではない）: 対象 `wiki/purposes/P-NNN.md` を削除し、`wiki/log.md` に `取り下げ（ユーザー判断）… ID P-NNN は欠番として再利用しない` を追記。ID は欠番として残す。被リンク `[[P-NNN]]` を grep して繋ぎ直す。`index.md` の行を削除。
- **書き換え**（前言撤回して新しい目的に立て直す）: 新しい `P-MMM` を起票し、frontmatter `revises: P-NNN`（＋本文 wikilink）で旧目的を指す。旧 P は `status: 棚上げ` にして残す（歴史性）。壁打ちでの書き換えは自由（確信度を上げるのでなく、目的文を磨く操作）。

## 守ること

- 一度に複数を量産しない。1本ずつ丁寧に。反証条件を言えない文は仮説ではない。
- **壁打ちだけで確信度を上げない**。精錬（言語化の解像度を上げる）は確信の根拠にならない。
