# /hand — 手中の鳥（制約C）を数える

エフェクチュエーション**①手中の鳥（Bird-in-hand）**の実装。いま手の内にある所与の資源を棚卸しし、制約レコード（C）として起票する。目的（P）は「何になりたいか」でなく「手の内に何があるか」から立ち上がる。数えた制約は目的に `grounded-in` で接地する。

> **共通規約**は [CLAUDE.md「スキル共通規約」](../../../CLAUDE.md)、理論（5原則）は [playbooks/effectuation.md](../../../playbooks/effectuation.md) が正典。以下は本スキル固有の手順。

## 手順

1. **現在案件を読む**（→ 共通規約1）。`wiki/constraints/` の既存 C と `wiki/purposes/` の既存 P を把握する。

2. **三分法で1つずつ数える** — `AskUserQuestion` で Effectuation の三分法に沿って具体的に問う（抽象的な理念でなく、いま実際に使える資源として）:
   - **自分は誰か**（type: 自分は誰か）— アイデンティティ・価値観・譲れない好み・向き不向き。
   - **何を知っているか**（type: 何を知っているか）— 知識・経験・専門・過去にやり切ったこと。
   - **誰を知っているか**（type: 誰を知っているか）— 頼れる人・つながり・コミュニティ。

3. **制約レコード C を起票する** — 意味のある単位ごとに `wiki/constraints/` の既存最大+1で `C-NNN` を採番し `templates/constraint.md` で作成する。`id` はファイル名と同じ接頭辞つき。`type` は上記3種のいずれか。本文に「これがどう目的の出発点になるか」を書く。`.gitkeep` が残っていれば削除してよい。

4. **目的に接地する（grounded-in）** — 既存の目的仮説（P）のうち、この制約から立ち上がっているものがあれば、その P の frontmatter `grounded-in` に `[[C-NNN]]` を追加し（配列）、本文「系譜・接地」にも wikilink を併記する（二重表現）。逆に「この制約から立ち上がりうる新しい目的」が見えたら `/surface`／`/formulate-purpose` に送る。
   - 確信度が高い（5以上）のに未接地の P は hwlint が `grounding-gap` で警告する。接地を促す。

5. **index と log を更新する** — `wiki/index.md` の「制約（手中の鳥）」表に1行追加（接地する目的があれば `[[P-NNN]]`）。`wiki/log.md` に追記:
   `## [YYYY-MM-DD] constraint | C-NNN <要約> → 起票（接地: P-NNN）`

## 守ること

- 制約は「制限」ではなく「出発点」。手段からの発想（Effectuation）を助ける。
- 抽象的な自己PRにしない。いま実際に使える具体的な資源として書く。
- 制約は確信度を持たない（Cレコードに confidence は無い）。
