# /lemonade — レモネード（想定外を逆手に取る）

エフェクチュエーション④。「酸っぱいレモン（想定外）を渡されたらレモネードを作れ」。想定外・驚き・偶然・断られを、**新しい目的の種やピボット**に変える。停滞して一本化しつつあるときに探索域を開き直す幅方向の一手でもある。

> **共通規約**は [CLAUDE.md「スキル共通規約」](../../../CLAUDE.md)、理論は [playbooks/effectuation.md](../../../playbooks/effectuation.md) が正典。以下は本スキル固有の手順。

## 手順

1. **現在案件を読む**（→ 共通規約1）。`wiki/reflections/`（内省 REF）の「驚き」、ACT 学習カードの「驚き・想定外」、`wiki/log.md` の断られ・想定外、`wiki/purposes/` の既存 P を棚卸しする。

2. **レモン（想定外）を集める** — 逆手に取れる素材を並べる:
   - REF・ACT に記録された**驚き**（予想と違ったこと）。
   - **断られ・失敗**（打診が通らなかった・試行が反証された）。
   - **偶然**（狙っていなかった反応・引き合い・接点）。
   - 素材が乏しければ、`/reflect` で最近の壁打ち・試行から驚きを拾うことを促す。

3. **逆手に取る（レモネードを作る）** — 各レモンについて `AskUserQuestion` で問う: 「この想定外は、どんな別の目的・別の相手・別の切り口を示唆しているか」。想定外を欠陥でなく**新しい方向の入口**として読み替える。

4. **新しい目的の種／ピボットに落とす**（承認を得てから）:
   - **新しい目的の種**: `wiki/purposes/` の既存最大+1で `P-NNN` を起票（`templates/purpose.md`、confidence 1-2・未検証、`/surface` と同じ低確信）。既存 P と別系統なら本文でその旨を、派生なら `derived-from` を書く。**壁打ち由来なので確信度は上げない**。
   - **ピボット**: 既存 P の方向を転換するなら `/decide`（`type: ピボット`）で記録し、新目的の `derived-from` に旧 P を書く。前言撤回して立て直すなら `/formulate-purpose` の `revises`。
   - どちらも `wiki/explore-log.md` に追記:
     `## [YYYY-MM-DD] 探索再起動 | <レモン> を逆手に → P-NNN 起票／P-MMM ピボット（幅方向）`

5. **index と log を更新する** — 目的仮説表を更新。`wiki/log.md` に追記:
   `## [YYYY-MM-DD] lemonade | <想定外> を逆手に → P-NNN 新規（確信度N/未検証）`

## 守ること

- 想定外を「計画からの逸脱＝失敗」でなく「新しい方向の材料」として扱う。
- 新しい種の確信度は低く正直に（壁打ち・想定外の読み替えは外界の証拠ではない）。育てるのは `/reflect`（試行）。
- 「探索の永久機関化」を避けるため、開いた種は増やしっぱなしにせず有望なものを `/afford`／`/quilt` で試行に送る。`explore-log.md` で探索の歴史を俯瞰できるようにする。
- 承認規律に従う（提案 → 承認 → 反映）。
