# tachiagari-wiki — キャリア目的形成Wiki

試行錯誤のループを通じて、**キャリアの目的・制約・使命が本人の中に事後的に立ち上がること**を支援する個人向け LLM-wiki。日々の壁打ち・試行を沈殿させて**確信を育て**、停滞したら**意図的に確信を揺らして探索域を開き直す**。目的が立ち上がったら目的達成型エージェントへ接続する。

`haru01/hypothesis-wiki`（スタートアップの仮説検証 CPF→PMF）を写像元に、キャリア目的形成ドメインへフォークしたもの。

## コアの考え方

- **目的は事前に設定や見つけるものではなく、試行錯誤で事後的に立ち上がる。** 最初の確信度は低くてよい。
- **二つの温度**: 確信度（目的の硬さ）と当事者性温度（関わりの深さ）は別軸（当事者性はフェーズ1では未実装・予約のみ）。
- **証拠の非対称性**: 確信度を上げてよいのは**外界に触れた証拠**（〈試行〉〈他者反応〉〈継続〉）に紐づくときだけ。壁打ち（エージェントとの対話）単体では上げない（壁打ち沼の防止）。
- **Effectuation（効果的思考の5原則）**: 予測でなく行動で未来を作る。①手中の鳥（手の内の資源＝制約Cから始める）／②許容可能な損失／③クレイジーキルト（パートナーの事前コミット）／④レモネード（想定外を逆手に）／⑤飛行機のパイロット（行動で制御）。理論は [playbooks/effectuation.md](playbooks/effectuation.md)。

## レコード種別（型）

`P` 目的仮説 ／ `C` 制約・手中の鳥 ／ `ACT` 試行 ／ `REF` 内省 ／ `DEC` 意思決定。型・関係・状態機械の正本は [ontology.yaml](ontology.yaml)（人間可読は [ontology.md](ontology.md)）。

## 使い方（フェーズ1・手動ループ）

操作は **Claude Code のスキル**（スラッシュコマンド）で行う。チャットに `/surface` のように打つと、Claude が「規律あるWikiの保守者」として問いを立て、規約に従って `projects/<slug>/wiki/` にレコードを起票・更新する。Claude は触媒・鏡であって、目的を代わりに決めない。

### はじめの一歩（初回）

1. **器を用意する — `/new-person`**
   自分用の案件（`projects/<slug>/`）を雛形から作る。まず試すだけなら、同梱の `self` 案件をそのまま使ってもよい。
2. **種を出す — `/surface`**
   Claude が「時間を忘れて没頭したこと」などを1問ずつ聞き、答えから**目的仮説（P）の種**を低い確信度（1〜2）で起票する。ここで確信を高めようとしなくてよい。
3. **反証可能にする — `/formulate-purpose`**
   種を「何が起きたら“やっぱり違った”と分かるか」（反証条件）まで1問ずつ精錬し、検証できる一文にする。

### 確信を育てるループ（2回目以降）

種ができたら、以下を回して**外界に触れながら確信を育てる**。骨格は**エフェクチュエーション5原則**（理論: [playbooks/effectuation.md](playbooks/effectuation.md)）。

1. **① 手中の鳥を数える — `/hand`**
   「自分は誰か／何を知っているか／誰を知っているか」を棚卸しして制約（C）にし、目的に `grounded-in` で接地する。目的は手の内の資源から立ち上げる。
2. **② 試行を設計する — `/afford`・③ `/quilt`**
   「結果に関わらず失ってよい範囲（許容可能な損失）」で最小の試行（ACT）を設計し（`/afford`）、乗ってくれるパートナーに事前コミットを取りにいく打診も組む（`/quilt`）。
3. **実際に試す（人が動く）**
   設計した試行を現実世界でやってみる。ここが確信の源泉。
4. **④ 取り込む — `/reflect`**
   壁打ち・試行・第三者反応の生データを取り込み、内省（REF）を書く。**確信度を上げられるのは外界に触れた証拠（trial／third-party）のときだけ**。壁打ち（dialogue）だけでは上げない（→ 壁打ち沼の防止＝証拠の非対称性）。
5. **停滞したら揺らす — ④ `/lemonade`・⑤ `/pilot`**
   驚き・想定外・断られを逆手に新しい種やピボットへ（`/lemonade`＝幅方向の探索）。逆に「予測・過信への逃避」「根拠なき確信」を突いて確信度を引き下げ、行動へ引き戻す（`/pilot`＝深さ方向の撹乱）。

```
①/hand → ②/afford・③/quilt → 実際に試す → ④/reflect（証拠なら確信度↑）
   ↑                                              │
   └────────── ④/lemonade（広げる）・⑤/pilot（揺らす）←┘
```

### セッションの例

```
あなた: /surface
Claude: （没頭したこと・放っておけないことを1問ずつ質問）
        → SELF-P-001「効くツールで新領域挑戦を支援する」を確信度2で起票

あなた: /hand
Claude: （手の内の資源を質問）→ 制約 C を4件起票し P-001 に接地

あなた: /afford
Claude: （許容できる損失を確認）→ 最小の打診 ACT を設計（未実施）

… 現実で打診してみる …

あなた: /reflect  （打診結果を sources/ に置いてから）
Claude: 内省を起票。他者が動いた証拠なので確信度 2→5 を提案 → 承認で反映
```

### スキル一覧

| 目的 | スキル |
|---|---|
| セットアップ | `/new-person` |
| 種出し・精錬（形成） | `/surface` `/formulate-purpose` |
| ①〜③ 試行の設計 | `/hand`① `/afford`② `/quilt`③ |
| Act→取り込み | `/reflect` |
| ④⑤ 探索・撹乱 | `/lemonade`④ `/pilot`⑤ |
| 支援（記録・俯瞰・保守） | `/decide` `/view` `/lint` |

## ツール

```bash
python3 tools/hwlint.py                # 決定論 lint（証拠非対称ルール含む）
python3 tools/gen_views.py list|board|relations   # ビュー生成
python3 tools/gen_ontology_doc.py      # ontology.yaml → ontology.md 再生成
python3 -m pytest tests/test_hwlint.py # テスト
```

Claude Code では Stop フックが自動で lint とビュー再生成を行う。初回に `git config core.hooksPath .githooks`。

## ロードマップ

- **フェーズ1（現在）**: 手動ループ＋日次沈殿。停滞検知は手動。
- **フェーズ2**: `/stall-check` で停滞検知を自動化（当事者性温度を本実装。確信度は自動で下げず本人承認を挟む）。
- **フェーズ3**: `/handoff` で立ち上がった目的を目的達成型エージェントへ接続。
- **フェーズ4**: 多人数運用への一般化。
