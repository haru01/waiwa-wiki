# エフェクチュエーション — 効果的思考のプレイブック

エフェクチュエーション（Effectuation, Sarasvathy）は、熟達した起業家が**予測できない状況**で使う意思決定の論理。未来を「予測して選ぶ」のでなく「行動して作る（制御する）」。このWikiのキャリア目的形成は、この効果的（effectual）ループで回す——**目的は事前に決めず、手中の手段から行動し、事後的に立ち上げる**。

5原則それぞれに対応スキルがあり、各スキルは冒頭でこのプレイブックを参照する。

## 因果的思考 vs 効果的思考

| 観点 | 因果的（causal・予測的） | 効果的（effectual・本ツールが採る） |
|---|---|---|
| 出発点 | 目的を決め、逆算して手段を選ぶ | いま手の内にある**手段**から始める（①手中の鳥） |
| 未来観 | 予測できる範囲で最適を選ぶ | **行動**を通じて未来を作る（⑤パイロット） |
| 投資判断 | 期待収益を最大化する | **失ってよい範囲**で踏み出す（②許容可能な損失） |
| 他者 | 競合を分析する | 乗ってくれる人と**組む**（③クレイジーキルト） |
| 想定外 | リスク＝避ける | 偶然・驚き＝**テコにする**（④レモネード） |

因果的思考は「予測できる未来」に強く、効果的思考は「予測できない未来」に強い。キャリアの目的形成は後者——だから本ツールは effectual に振る。

## 5原則

### ① 手中の鳥（Bird-in-hand）→ `/hand`
- **定義**: 目的から逆算せず、いま手の内にある手段——**自分は誰か／何を知っているか／誰を知っているか**——から始める。
- **Wiki**: `/hand` が制約 C を数え、目的 P に `grounded-in` で接地する。
- **問い**: いま手の内に何があるか。それから何が作れるか。

### ② 許容可能な損失（Affordable loss）→ `/afford`
- **定義**: 「うまくいけばどれだけ得か（期待収益）」でなく「**失敗しても失ってよいのはどこまでか**（許容損失）」で投資を決める。だから予測なしでも小さく踏み出せる。
- **Wiki**: `/afford` が次の試行を許容損失で設計する。予測的な「大きく賭ける／もっと確証を得てから」を、失ってよい範囲の最小の一歩に戻す。産物は試行 ACT（テストカードに許容損失を明記）または `/decide` の許容損失欄。
- **問い**: 結果に関わらず失ってよい時間・金・評判・機会はどこまでか。その範囲でできる最小の一歩は。

### ③ クレイジーキルト（Crazy quilt）→ `/quilt`
- **定義**: 競合分析より、自ら手を挙げて**コミットする人とパートナーシップ**を組む。パートナーの事前コミットが、進む方向と使える資源を共に決める（目的すら共創されうる）。
- **Wiki**: `/quilt` がパートナー・協力者を洗い出し（既存 C「誰を知っているか」＋新規 C）、**事前コミット**（時間・紹介・小さな約束）を取りにいく打診を設計する。得たコミットは第三者反応（third-party）証拠として `/reflect` が確信度に反映する。
- **問い**: 誰が一緒に乗ってくれるか。いまどんな小さなコミットをもらえるか。

### ④ レモネード（Lemonade）→ `/lemonade`
- **定義**: 「酸っぱいレモン（想定外）を渡されたらレモネードを作れ」。想定外・偶然・断られを避けず、**テコにして新しい方向**を作る。
- **Wiki**: `/lemonade` が内省 REF の「驚き」・断られ・偶然を棚卸しし、逆手に取って新しい目的の種（→ P 起票 confidence 1-2）やピボット（→ `/decide`）に変える。`explore-log.md` に `探索再起動` を追記。
- **問い**: 最近の想定外・驚き・断られは何か。それを逆手に取ると何が見えるか。

### ⑤ 飛行機の中のパイロット（Pilot-in-the-plane）→ `/pilot`（統合思想）
- **定義**: 未来は予測するものでなく、**行動を通じて制御する**もの。人間の行動こそが結果を生む主因。予測に逃げず、動いて確かめる。①〜④を束ねる worldview。
- **Wiki**: これがループ全体の思想的支柱であり、**証拠の非対称性の下げ側**の根拠でもある（外界の行動で裏打ちされない確信は下げる）。`/pilot` が予測的ドリフト（完璧な計画／市場予測／「もっと考えてから」／壁打ちだけの確信）を突き、行動で確かめる次の一手へ引き戻す。根拠なき確信度は根拠づけて引き下げる。
- **問い**: それは予測か、行動で確かめられることか。いま動いて確かめられる最小の一歩は。

## このWikiでの効果的ループ

```
①手中の鳥(/hand) ──┐
                    ├─→ ②許容損失(/afford)＋③パートナー(/quilt) で試行を設計
                    │        ↓（人が現実で試す）
                    │   /reflect で観測・確信度更新（証拠の非対称性）
                    │        ↓（想定外・驚き）
                    └───  ④レモネード(/lemonade) で新しい目的の種／ピボット
                             ↕
                    ⑤パイロット(/pilot)：予測への逃避と根拠なき確信を突き、行動へ引き戻す
```

目的（P）は最初から確信度が高い必要はない。**手段から行動し、外界の反応と偶然を織り込みながら、目的を事後的に立ち上げる**。予測（壁打ち）で確信を膨らませず、行動（試行・他者反応）で確信を育てるのが効果的思考の核心であり、本ツールの証拠の非対称性と一致する。
