# /afford — 許容可能な損失で次の一手を設計する

エフェクチュエーション②。「うまくいけばどれだけ得か」でなく「**失敗しても失ってよいのはどこまでか**」で投資を決める。予測なしに小さく踏み出せる状態を作る。診断（予測的な賭け方への偏りを突く）＋提案（許容損失内の最小試行を設計）。

> **共通規約**は [CLAUDE.md「スキル共通規約」](../../../CLAUDE.md)、理論は [playbooks/effectuation.md](../../../playbooks/effectuation.md) が正典。以下は本スキル固有の手順。

## 手順

1. **現在案件・対象を読む**（→ 共通規約1）。対象の目的 P（`wiki/purposes/`）と、既にある試行 ACT・手中の鳥 C を把握する。

2. **予測的思考を診断する** — 対象の計画・言い分に次の兆候がないか突く（引用つきで）:
   - 「うまくいけば大きい」（期待収益で正当化している）／「もっと確証を得てから」「完璧に準備してから」（予測に逃げている）／一発勝負・後戻りできない賭け方になっている。
   - これらは因果的（予測的）思考の兆候。効果的思考では**下振れ（失う量）**を先に見る。

3. **許容損失を確定する** — `AskUserQuestion` で1問ずつ、次の試行について「**結果に関わらず失ってよい量**」を洗い出す:
   - 時間（何時間／何日まで）／お金（いくらまで）／評判・関係（誰にどう思われても構わない範囲）／機会（この間できない他のこと）。
   - 「失ったら痛いが致命的でない」線を引く。ここを超える賭けは却下・分割する。

4. **許容損失内の最小の試行を設計する** — 許容損失に収まる**最小の一歩**を提案する。産物は次のいずれか（承認を得てから作成）:
   - **試行 ACT の起票**（`type` は 打診／やってみる／面談／観察）: `templates/activity.md` のテストカードに、目的・方法・指標・成功基準に加えて**許容損失（今回失ってよい量）**を明記する。`purposes` に対象 P、`riskiest-assumption` に「この試行で崩れたら目的が崩れる一点」。learning card は空（実施後に `/reflect` が埋める）。
   - **`/decide` への引き渡し**: コミット規模の判断なら、DEC 本文に許容損失を刻んで `/decide` で記録する。
   - この時点では確信度を動かさない（設計であって観測でない）。確信度を上げるのは実施後の `/reflect`。

5. **index と log を更新する** — ACT を起票したら `wiki/log.md` に追記:
   `## [YYYY-MM-DD] <ACTのtype> | ACT-NNN <要約>（許容損失: …）→ 試行設計（未実施）`

## 守ること

- 期待収益（上振れ）で GO を出さない。**下振れ（失ってよい量）**で GO/分割/却下を決める。
- 「もっと考えてから」を許容損失内の行動に変える。予測の精度を上げるより、失ってよい範囲で動いて確かめる（→ ⑤ `/pilot` と同じ思想）。
- このスキルは確信度を動かさない（試行の設計まで）。実施結果の取り込みと確信度更新は `/reflect`。
- 承認規律に従う（提案 → 承認 → 反映）。
