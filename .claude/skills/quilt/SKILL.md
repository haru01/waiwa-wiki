# /quilt — クレイジーキルト（パートナーと事前コミット）

エフェクチュエーション③。競合を分析するより、**自ら手を挙げてコミットしてくれる人**と組む。パートナーの事前コミットが、進む方向と使える資源を共に決める。診断（一人で抱えていないか）＋提案（誰に・どんな小さなコミットを求めるか）。

> **共通規約**は [CLAUDE.md「スキル共通規約」](../../../CLAUDE.md)、理論は [playbooks/effectuation.md](../../../playbooks/effectuation.md) が正典。以下は本スキル固有の手順。

## 手順

1. **現在案件・対象を読む**（→ 共通規約1）。対象の目的 P と、既存の制約 C（特に `type: 誰を知っているか`）を把握する。

2. **一人で抱えていないか診断する** — 目的を自分だけで完結させようとしていないか、「まず完成させてから見せる」になっていないかを突く。効果的思考では**早くパートナーを巻き込み**、コミットで方向を共に決める。

3. **パートナー候補を洗い出す** — `AskUserQuestion` で「この目的に乗ってくれそうな／既に関心を示している人・場」を具体的に挙げる。既存 C（誰を知っているか。例 [[SELF-C-004]]）を起点にする。新しいつながりが出たら制約 C（`type: 誰を知っているか`）として起票する。

4. **事前コミットの打診を設計する** — 各候補に対し「**いま求められる最小のコミット**」を設計する。大きな賛同でなく、時間・紹介・小さな約束・次に会う約束など、相手が実際に一歩踏み出す具体的な依頼。
   - 産物: **打診 ACT の起票**（`type: 打診`、`templates/activity.md`）。テストカードに、誰に・何を頼むか（方法）と**成功基準＝相手が具体的なコミットをするか**を書く。`purposes` に対象 P。learning card は空（実施後 `/reflect`）。
   - お世辞・「いいね」はコミットでない（[playbooks/interviewing.md](../../../playbooks/interviewing.md)）。**時間・手間・紹介など相手がコストを払う**約束を成功基準にする。

5. **得たコミットは証拠として `/reflect` へ** — 実際に打診してコミットが得られたら、その記録を `sources/`（**種別 third-party**）に置き `/reflect` で取り込む。第三者が実際に動いた証拠（〈他者反応〉）として確信度を上げられる。

6. **index と log を更新する** — C を起票したら「制約」表に追加。打診 ACT を起票したら `wiki/log.md` に追記:
   `## [YYYY-MM-DD] 打診 | ACT-NNN <誰に何を打診> → 試行設計（未実施）`

## 守ること

- 競合分析に時間を使うより、乗ってくれる人を1人見つけて小さくコミットしてもらう。
- コミットは**相手がコストを払う**具体的な約束に限る（お世辞・関心表明は〈壁打ち〉相当で確信度を上げない）。
- このスキルは確信度を動かさない（打診の設計まで）。コミット結果の取り込みは `/reflect`（third-party）。
- 承認規律に従う（提案 → 承認 → 反映）。
