# 効くツールでエンジニアの新領域挑戦を支援する

## 目的仮説文（反証可能な形式で）

> 私は「Claude Code で効くツール／プロトタイプを作れる」という手中の鳥から、新しいエリア（DDD・仮説検証など）に挑戦するエンジニアが、**私の作ったツールを使って実際にその挑戦を前に進められる**状態を目的とする。

### 反証条件（成否は「相手の前進」で判定する）

- **立ち上がる（支持）シグナル**: 支援したエンジニアが、私のツールを使って新領域の課題を**自力で前に進めた**（できれば繰り返し使う・他の人にも広がる）。
- **揺らぐ（反証）シグナル**: ツールは動くが、誰も新領域を前に進めない／使われない／自分が「動いた」で満足して相手の前進を確かめないまま終わる。

### 精錬メモ（`/formulate-purpose`）

- **動機の芯と成否条件のねじれ（重要・要監視）**: 満足の芯は「作って動かすこと自体」＝自己完結的（本人選択）。一方で、この目的の**成否条件は外向き（相手が前に進んだか）に置くと本人が決めた**。この2つはねじれている。
  - 生産的な緊張になりうる（内発的な作り手のエネルギーで外向きの成果を出す）。
  - 同時に**偽の収束のリスク**でもある: 「動いた」で満足が満たされてしまい、相手の前進を確かめないまま終わりやすい。→ `/pilot`（⑤飛行機のパイロット）の標的として明記。当事者性温度の判定はフェーズ1では行わない（記録のみ）。
- 芯（作ったものが効くこと）は軸として保持。芯を外向き成否条件で測る、という形に締めた。

## 前提

- エンジニアが新しいエリアに挑戦するときの障壁が実在し、ツールで下げられる。
- 「効くツールを作る」こと自体が、本人にとって（人の成長支援そのものより）強い動機になる。
