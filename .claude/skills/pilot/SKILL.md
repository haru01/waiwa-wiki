# /pilot — 飛行機の中のパイロット（予測を突き、行動へ引き戻す）

エフェクチュエーション⑤。未来は予測するものでなく**行動を通じて制御する**もの。予測・過信への逃避を突き、根拠なき確信度を根拠づけて引き下げ、**行動で確かめる次の一手**へ引き戻す。①〜④を束ねる統合的な規律であり、証拠の非対称性の**下げ側**（外界の行動で裏打ちされない確信は下げる）を担う。

引き下げは**根拠づけてのみ**行う。監査材料を新規 source（種別 dialogue）に残し、`self-reflection` の試行レコード（ACT）に紐づけて**提案 → ユーザー承認 → 反映**する。挑発のために数値を無根拠に動かさない（下げるのも上げると同じ規律が要る）。露出した新しい探索域は `/lemonade` へ委譲する。

> **共通規約**は [CLAUDE.md「スキル共通規約」](../../../CLAUDE.md)、理論は [playbooks/effectuation.md](../../../playbooks/effectuation.md) が正典。以下は本スキル固有の手順。

## 手順

1. **現在案件を読む**（→ 共通規約1）。[playbooks/interviewing.md](../../../playbooks/interviewing.md) の確証バイアス・証拠の非対称性の心得を読む。

2. **スコープを決める（`AskUserQuestion`）**:
   - **①狙い撃ち**: 指定した特定目的（`P-NNN`）を深く突く。
   - **②全体スイープ**: `wiki/index.md`・`wiki/purposes/` に加え **`wiki/views/relations.md`**（関係グラフ・バックリンク索引・接地フィット。無ければ `python3 tools/gen_views.py relations` で生成）を読み、危険標的を優先する:
     - **偽の収束**: 確信度が高い（または高原で動かない）のに外界の証拠が薄い／試行が枯れている（hwlint `untested-focus`）。
     - **未接地**: 確信度が高いのに `grounded-in`（手中の鳥C）が空（hwlint `grounding-gap`）。
     - **孤立**: バックリンク被参照ゼロなのに高確信＝放置された高確信。
     - **壁打ち沼**: 確信度履歴の根拠が〈壁打ち〉ばかりで〈試行〉以上が乏しいのに数字が上がっている。
   - 対象ごとに **目的仮説本文＋確信度履歴＋紐づく `[[ACT]]`＋参照 `sources/`（種別タグ）＋関係近傍** を必ず読んでから突く。

3. **予測への逃避と過信を突く（批判的監査）** — 該当を具体的な引用つきで指摘する:
   - **予測的ドリフト**（このスキル固有の中核）: 「完璧な計画／市場を予測してから／もっと確証を得てから／今は考えている段階」——**行動で確かめられることを予測に置き換えていないか**。予測は制御でない。
   - **証拠の非対称性違反**: 確信度を上げた根拠が〈壁打ち〉（対話のみ）止まりで、外界に触れた証拠（〈試行〉〈他者反応〉〈継続〉）が無いのに数字が高くないか。
   - **偽の収束**: 確信は高いが腰が入っているか。楽な目的に飛びついていないか（※フェーズ1は定性的に問う。当事者性温度は判定に使わない）。
   - **確証バイアス**: 支持証拠ばかりでないか。「この目的が“違う”証拠を3つ挙げられるか」。
   - **反証の未反映**: log・学習カード・驚きに反証があるのに確信度に反映されていないか。
   - **反証可能性の欠如／前提の脆さ／サンプルの弱さ**: 何が起きれば揺らぐかが曖昧でないか。崩れたら目的ごと崩れる前提はどれか。n=1・一度きりに依存していないか。
   - **連鎖構造（関係グラフ）**: 上流目的の引き下げが `leads-to` 下流に伝播していない／派生元が揺らいだのに派生先が高確信／未接地・孤立の高確信ノード。
   > 指摘は手段であって目的ではない。名前を言うだけで終えず、**なぜ危険か／どう行動で確かめるか**まで示す。監査者自身の確証バイアスも点検する。

4. **監査材料を sources/ に記録する（新規・不変）** — `sources/YYYY-MM-DD-pilot-<topic>.md` を新規作成。冒頭に必ず:
   > ⚠️ **反証監査（自己批判）メモ。種別: dialogue。新規の実観測証拠ではなく、既存記録への批判的再解釈。これ自体は確信度を上げる根拠にならない。**

   対象ごとに **予測への逃避の兆候／危うい前提／根拠不足の具体指摘（どの履歴行のどのタグが弱いか）／別解釈** を構造化して書く。**一度置いた source は編集しない**。

5. **self-reflection の試行レコード（ACT）を作る** — `wiki/activities/` の既存最大+1で `ACT-NNN` を採番し `templates/activity.md` に従い `type: self-reflection` で作成する。
   - 本文冒頭に `対象目的: [[P-NNN]]` と `監査材料: sources/YYYY-MM-DD-pilot-<topic>.md`（相対mdリンク）。frontmatter `purposes` は接頭辞つき配列。
   - **テストカード**（事前登録・後から書き換えない）: 成功基準＝**現在の確信度が外界に触れた証拠で正当化できなければ引き下げ、正当化できれば据え置き（＝反証耐性の確認）**。`riskiest-assumption` に監査が崩しに行く前提を一文。
   - **学習カード**: 学びの要点／事実（記録に何が書かれているか引用）／解釈（どこが予測への逃避か・根拠不足か）／驚き・想定外／確信度の更新テーブル／次のアクション。引き下げたら `outcome: 是正`、崩れず据え置きなら `outcome: 判断保留`。

6. **確信度の引き下げを提案する（提案 → 承認 → 反映）** — 対象ごとに **現在値 → 提案値・提案ステータス・理由（発火した観点）** を提示する。**必ずユーザーの承認を得てから反映する**。
   - **引き下げは、現在値が外界に触れた証拠で正当化できないと示せる場合に限る**（例: 6だが根拠が〈壁打ち〉のみ→3、反証が出ているのに据え置き→引き下げ＋status変更）。
   - 揺さぶっても崩れなければ**据え置き**とし1行記録する（反証耐性の確認）。
   - **このスキルは確信度を下げる／据え置くのみ。机上監査で確信度を上げない**（上げ直しは実証拠を得て `/reflect` で行う）。
   - 承認後、`wiki/purposes/P-NNN.md` の frontmatter `confidence`/`status` を更新し、確信度履歴テーブルに1行追記（活動列に `[[ACT-NNN]]`、根拠列の先頭に `〈壁打ち〉根拠不足のため減点` 等）。

7. **行動で確かめる次の一手へ引き戻す** — 監査の payoff は「下がった数字を眺めて終わる」ことでない。ACT の「次のアクション」に:
   - **行動で確かめる最小の一歩** → `/afford`（許容損失内の試行）・`/quilt`（パートナーに打診）へ送る（予測でなく行動で確信を取りにいく）。
   - **露出した別解釈・新しい探索域** → `/lemonade` へ送る（想定外・別方向を新しい目的の種に）。

8. **index・log・explore-log を更新する** — `wiki/index.md` の該当行を更新。`wiki/log.md` に追記:
   `## [YYYY-MM-DD] pilot | ACT-NNN 反証監査 → P-NNN 確信度X→Y（根拠不足）／P-MMM 据え置き（反証耐性確認）`
   `wiki/explore-log.md` に追記:
   `## [YYYY-MM-DD] 意図的引き下げ | P-NNN 確信度X→Y（予測への逃避／根拠不足）／次の行動: …`

## 守ること

- 監査は**記録への批判であって人格攻撃ではない**。欠陥を突くなら必ず**別解釈か次の行動**を添える。
- 確信度を**下げるのも上げると同じく規律が要る**。根拠なく動かさない。挑発目的で数値操作しない。
- **机上監査で確信度を上げない**（下げる／据え置くのみ）。上げ直しは `/reflect`（外界の証拠）。
- `sources/` は新規追加のみ・不変。監査メモ冒頭に「新規証拠でない・種別 dialogue」を必ず明記する。
- 予測を突いたら、必ず**行動で確かめる道**（`/afford`・`/quilt`）か**探索を開く道**（`/lemonade`）に接続する。
- 反映は必ず承認を得てから（承認規律は共通規約5）。
